Rapport d'Audrey Leconte

Mon application Candy Maths est destinée aux enfants en classe de CP pour l'apprentissage des additions. Pour faciliter la visualisation des nombres et rendre plus ludique les mathématiques, la calculatrice affiche le résultat à l'aide de paquets de bonbons. Pour inciter les enfants à se servir régulièremment de l'application, un quiz y est intégré, proposant des questions de culture générale.

Principes UX - Joel Marsh

Pour la calculatrice, le résultat est affiché à l'aide de bonbons colorés. J'ai suivi les conseils de l'article Colour de Joel Marsh (lien vers article). Ainsi, plus le bonbon est foncé, plus il représente un grand nombre de bonbons.

Presentation innovation

L'innovation de cette application est l'affichage du résultat d'une addition à l'aide de bonbons. La visualisation et la compréhension des nombres peuvent être difficiles pour des enfants. Cette calculatrice leur permet donc de mieux appréhender les additions et de façon plus ludique. De plus, le quiz intégré à cette application leur permet d'améliorer leur culture générale tout en les incitant à revenir régulièremment s'entraîner. En effet, le quiz propose une question différente à chaque ouverture de l'application.

Nommage des tests

Pour nommer mes tests, j'ai suivi la méthode décrite dans le document Tests unitaires en Java avec JUnit 5 disponible dans le cours de Génie logiciel et patrons de conception sur Moodle. J'ai donc utilisé la notation snake_case et le patron should-when : should_ComportementAttendu_when_EtatTesté.

Effort fonctionnel pour les tests

Pour le modèle de ma calculatrice, j'utilise la libairie exp4j. J'ai donc réalisé différents tests sur cette librairie :

  • tests des opérations de base (+, -, /, *, %)
  • tests de calculs avec des variables
  • tests avec des fonctions mathématiques
  • tests avec des multiplications implicites
  • tests avec des constantes numériques
  • tests avec des fonctions personnalisées
  • tests avec une division par zéro
  • tests pour vérifier la validité d'une expression.

Technique de tests avancée

J'ai généré un test coverage report en activant le plugin JaCoCo. Le test coverage report correspond au fichier app/build/reports/tests/testDebugUnitTest/index.html.

Responsive

Mon application est responsive puisqu'elle peut être utilisée sur tablette comme sur téléphone. Pour ce faire, pour chaque activité (page d'accueil, calculatrice et quiz), il y a deux fichiers .xml pour les téléphones et deux pour les tablettes (sw600dp).

Vertical - Horizontal

Mon application peut s'utiliser à l'horizontale comme à la verticale. Pour chaque activité (page d'accueil, calculatrice et quiz), pour chaque type de support (téléphone, tablette), il y a un fichier .xml pour la version horizontale (land) et un fichier .xml pour la version verticale.

Installation sur téléphone et tablette

J'ai installé mon application sur mon téléphone et sur ma tablette.

Ressources pour constantes

Les fichiers correspondant à mes ressources pour mes constantes sont :

  • colors.xml : pour les couleurs de mon application
  • strings.xml : pour les textes des boutons et les descriptions des éléments.

Usage pertinent Layout

J'ai utilisé des Layout en fonction de mes besoins :

  • TableLayout : pour organiser les boutons de ma calculatrice
  • GridLayout : pour afficher plusieurs images de bonbons en fonction du résultat.

Volley

Je me suis servie de la bibliothèque Volley pour effectuer des requêtes avec mon API.

JSON

Les réponses fournies par mon API sont au format JSON (exemple réponse API). Je me suis servie de différentes classes telles que JSONArray et JSONObject pour analyser et me servir des données.

Config CICD (main forcee)

J'ai créé un pipeline sur GitHub en ajoutant un fichier .yml dans le dossier .github/workflows/ de mon projet.

Recherches personnelles

Dans un premier temps, j'ai effectué des recherches sur le travail et les besoins des enfants en classe de CP. J'ai ensuite choisi de créer une calculatrice avec des bonbons pour aider les enfants à se représenter les nombres. J'ai également effectué beaucoup de recherches pour trouver une API, car la plupart étaient soit en anglais, soit payantes, soit inadaptées. J'ai cherché des PNG pour assembler et créer mes images de légendes et de fonds. Enfin, j'ai lu plusieurs articles pour résoudre des problèmes d'implémentation, tels que ceux relatifs à l'architecture MVVM ou aux fichiers .xml pour gérer les versions horizontales et verticales des activités.

MVP

Pour ma calculatrice, j'ai essayé d'implémenter l'architecture MVVM. Malheureusement, malgré de nombreuses recherches et plusieurs tests d'implémentation (avec des RecyclerView et GridLayoutManager), je n'ai pas réussi à le mettre en place car la gestion des images avec MVVM et mon GridLayout était trop compliquée. J'ai donc finalement implémenté l'architecture MVP.

Critique travail

L'API utilisée pour le quiz n'est pas spécifiquement conçue pour les enfants. Les questions sont néanmoins, pour la plupart, à la portée des enfants. Pour l'architecture de la calculatrice, j'aurais aimé utiliser MVVM mais je n'ai pas réussi à l'implémenter à cause de la gestion des images.

Github

J'ai commencé mon travail avec Gitlab, mais suite à la cyberattaque de l'ENSICAEN, j'ai déplacé mon projet sur Github. J'ai donc fait un github pages et mon historique des commits contient tous les commits depuis le début.

Precisions sur mon travail

Pour la calculatrice, chaque opérande dans le calcul doit être un nombre inférieur ou égal à 99. Ensuite, les boutons des chiffres deviennent gris si l'enfant doit ajouter le signe + ou si le calcul est terminé. Le résultat maximum est donc 198 bonbons. J'ai également changé le logo par défaut de mon application pour mettre celui qui se trouve tout en haut de cette page : Candy Maths. J'ai aussi ajouté des contentDescription pour les éléments dans mes fichiers .xml afin de permettre aux enfants ayant des déficiences visuelles de comprendre les éléments grâce aux outils d'accessibilité, comme les lecteurs d'écran. Pour le quiz, lorsque l'enfant clique sur une réponse, un message apparaît pour confirmer si la réponse est correcte ou non.
